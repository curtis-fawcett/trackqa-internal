# TrackQA (Internal)

Internal QA and debugging workflow dashboard for tracking bugs, refactors, investigations, and verification.

## Overview

TrackQA is a lightweight internal tool designed to help manage and document the debugging process across projects. It focuses on clarity, traceability, and structured investigation workflows.

## Features

- Kanban-style ticket management
- Bug, refactor, and technical debt tracking
- Investigation details (reproduction steps, root cause, resolution)
- Debug timeline logging
- QA verification tracking
- Tag-based organization
- Local storage persistence

## Tech Stack

- React
- React Router
- LocalStorage (no backend yet)

## Purpose

This tool was built to improve how debugging and QA workflows are tracked internally, making it easier to:

- Understand issues quickly
- Document root causes clearly
- Track fixes and verification steps
- Maintain a clean debugging history
